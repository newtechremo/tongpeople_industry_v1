### 1.3 URL 패턴
```
/risk-assessment/initial/{assessmentId}?status=completed
```

## 5. 사용자 시나리오

### 시나리오 1: 작업종료된 위험성평가 조회

1. 사용자가 위험성평가 목록에서 `작업종료` 상태의 문서 클릭
2. 상세보기 화면 진입
3. 헤더에 `[최초] 작업명` 및 `작업종료` 뱃지 표시
4. 결재 라인은 읽기 전용으로 표시 (수정 아이콘 없음)
5. 위험요인 카드 목록 조회

### 시나리오 2: PDF 출력

1. 사용자가 `출력` 버튼 클릭
2. 드롭다운 메뉴에서 `PDF로 저장` 선택
3. 파일 다운로드 시작

### 시나리오 3: 문서 삭제

1. 사용자가 `삭제` 버튼 클릭
2. 삭제 확인 모달 표시
3. `삭제` 버튼 클릭
4. 삭제 완료 후 목록 화면으로 이동
5. 목록에서 해당 문서 제거됨

### 시나리오 4: 수정 시도 (차단됨)

1. 작업종료 상태에서 수정 기능 접근 시도
2. "수정 권한이 없습니다." 모달 표시
3. 확인 클릭 후 모달 닫힘

# Gemini 프롬프트 - 정보 입력 페이지 예시 생성

아래 프롬프트를 Gemini에 복사-붙여넣기 하세요.

---

## 프롬프트

```
React Native + NativeWind(Tailwind CSS)로 모바일 앱 화면을 구현해주세요.

## 요청사항
"정보 입력" 화면을 구현해주세요. 이 화면은 회원가입 플로우의 일부로, 사용자의 기본 정보를 입력받는 폼 화면입니다.

## 디자인 시스템 규칙

### ⚠️ 중요: 라이트 모드 디자인입니다
- 이 앱은 **라이트 모드**입니다. 다크 모드가 아닙니다.
- 화면 배경: 흰색(#FFFFFF) 또는 연한 회색(#F9FAFB)
- 입력 필드 배경: **반드시 흰색(#FFFFFF)** 사용
- 검은색/어두운 배경의 입력 필드는 절대 사용하지 않습니다.

### 1. 색상
| 용도 | 색상 | HEX |
|------|------|-----|
| Primary | Orange 500 | #F97316 |
| Primary Dark | Orange 600 | #EA580C |
| Primary Light | Orange 50 | #FFF7ED |
| **화면 배경** | White | **#FFFFFF** |
| **입력 필드 배경** | White | **#FFFFFF** |
| Text Primary | Slate 800 | #1E293B |
| Text Secondary | Slate 600 | #475569 |
| Placeholder 텍스트 | Slate 400 | #94A3B8 |
| Border Default | Slate 300 | #CBD5E1 |
| Border Disabled | Slate 200 | #E2E8F0 |
| Background Disabled | Slate 100 | #F1F5F9 (연한 회색) |
| Error | Red 600 | #DC2626 |
| Success | Green 600 | #16A34A |

### 2. 입력 필드 상태 (6단계)

**⚠️ 모든 입력 필드의 배경은 흰색(#FFFFFF)입니다. 검은색 배경을 사용하지 마세요.**

| 상태 | 테두리 | 배경 | 텍스트 |
|------|--------|------|--------|
| 입력 전 (Default) | #CBD5E1 (1px) | **#FFFFFF (흰색)** | placeholder: #94A3B8 (회색) |
| 입력 중 (Focused) | #F97316 (2px) + ring | **#FFFFFF (흰색)** | #1E293B (검은색 텍스트) |
| 입력 후 (Filled) | #CBD5E1 (1px) | **#FFFFFF (흰색)** | #1E293B (검은색 텍스트) |
| 입력 불가 (Disabled) | #E2E8F0 (1px) | #F1F5F9 (연한 회색) | #94A3B8 (회색) |
| 에러 (Error) | #DC2626 (2px) | **#FFFFFF (흰색)** | #1E293B + 하단 에러 메시지 |
| 성공 (Success) | #16A34A (1px) | **#FFFFFF (흰색)** | #1E293B + 체크 아이콘 |

### 3. 컴포넌트 스펙

**헤더:**
- 높이: 56px
- 배경: #F97316 (오렌지)
- 텍스트: #FFFFFF (흰색), 18px SemiBold
- 뒤로가기 아이콘: 흰색

**입력 필드:** ⚠️ 배경은 반드시 흰색!
- 높이: 52px
- 라운딩: 12px (rounded-xl)
- **배경색: #FFFFFF (흰색)** ← 중요!
- **테두리: #CBD5E1 (연한 회색 선)**
- 폰트: 16px Regular, 텍스트 색상 #1E293B
- placeholder 텍스트: #94A3B8 (회색)
- 라벨: 14px Medium, #475569, 하단 8px 간격

**버튼 (Primary):**
- 높이: 52px
- 라운딩: 12px (rounded-xl)
- 배경: linear-gradient(#F97316 → #EA580C)
- 비활성: #E5E7EB 배경, #9CA3AF 텍스트

**라디오 버튼:**
- 크기: 24px (외부), 12px (내부 점)
- 선택됨: 테두리 #F97316, 내부 점 #F97316
- 미선택: 테두리 #CBD5E1

**드롭다운:**
- 입력 필드와 동일한 스타일
- 우측에 ChevronDown 아이콘 (#94A3B8)
- 터치 시 BottomSheet 열림

### 4. 폰트
- Regular: Pretendard-Regular
- Medium: Pretendard-Medium
- SemiBold: Pretendard-SemiBold
- Bold: Pretendard-Bold

### 5. 간격
- 화면 좌우 패딩: 16px
- 필드 간 간격: 16px
- 라벨-입력 간격: 8px
- 섹션 간 간격: 24px

## 화면 구성

### 헤더
- 제목: "정보 입력"
- 뒤로가기 버튼 있음 (왼쪽)
- 오렌지 배경 (#F97316)

### 선등록 배너 (조건부)
```
┌─────────────────────────────────────┐
│ ✓ 관리자가 등록한 정보를 불러왔습니다   │
└─────────────────────────────────────┘
```
- 배경: #DCFCE7 (Green 100)
- 테두리: #BBF7D0 (Green 200)
- 아이콘: 체크, #16A34A
- 텍스트: #15803D

### 입력 필드 목록

1. **이름** (필수)
   - placeholder: "이름을 입력해주세요"
   - 검증: 2자 이상

2. **생년월일** (필수)
   - placeholder: "예: 19850315"
   - 키보드: 숫자 (numeric)
   - 최대 8자
   - 힌트 텍스트: "예: 1985년 3월 15일 → 19850315"

3. **휴대폰번호** (읽기 전용)
   - 값: "010-1234-5678"
   - 라벨 옆에 "✓ 인증완료" 표시 (녹색)
   - disabled 상태 (회색 배경)

4. **이메일주소** (선택)
   - 라벨: "이메일주소 (선택)"
   - placeholder: "example@email.com"
   - 키보드: email-address

5. **성별** (필수, 라디오)
   - 옵션: 남성, 여성
   - 가로 배치 (flex-row)

6. **국적** (필수, 드롭다운)
   - 기본값: "대한민국"
   - 옵션: 대한민국, 중국, 베트남, 네팔, 기타...

7. **소속 팀** (필수, 드롭다운)
   - placeholder: "소속 팀을 선택해주세요"

8. **직책/직종** (필수, 드롭다운)
   - placeholder: "직책/직종을 선택해주세요"

### 하단 버튼
- 텍스트: "다음"
- 모든 필수 필드 입력 시 활성화
- 하단 고정, SafeArea 고려

## 추가 요청사항

1. 스크롤 가능한 레이아웃 (KeyboardAvoidingView 포함)
2. 포커스 시 입력 필드 테두리 색상 변경 (#F97316)
3. 에러 상태 예시 1개 포함 (이름 필드 - "이름은 2자 이상 입력해주세요")
4. TypeScript로 작성
5. 상태 관리는 useState 사용
6. 컴포넌트를 분리하지 않고 단일 파일로 작성

## 결과물
React Native + TypeScript 코드 전체를 제공해주세요.
주석으로 각 섹션을 설명해주세요.
```
